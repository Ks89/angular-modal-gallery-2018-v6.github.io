ModalGallery

The ModalGallery component is a collection of images. It displays images as thumbnails in a single row. Clicking on one of them, the image will be showed in full screen in a modal way.

In addition to this, ModalGallery provides navigation between modal images with both mouse and keyboard and it can enable/disable download features. To conclude, it is also possible to configure descriptions and to add a button to navigate to an external URL.


Selector modal-gallery
Exported as modalGallery

Inputs

modalImages
Type: Array<Image> | Observable<Array<Image>>
An array of Image objects or an Observable of it, that represent both the thumb gallery and the modal images. In fact, an Image is an object with both thumb and image source paths. For more info check Image.
downloadable
Type: booleanoptional

If true, you'll be able to download the current modal image always with keyboard shortcut (ctrl+s or cmd+s), but the download button will be displayed only if showDownloadButton will be true. Otherwise, all download features will be disabled. downloadable=false by default

Please be careful: on SmartPhones and Tablets you can't use keyboard shortcuts, so to download images, please enable both downloadable and showDownloadButton to add the download button.
description
Type: Description interfaceoptional

Interface to specify a custom description either a full string or an object with string properties. The first one is the entire string (for instance 'Description of Image 1/2 - My family - bla bla'), the second one is an object with these properties imageText (default is 'Image'), numberSeparator (default is '/'), beforeTextDescription (default is ' - ').

To customize part of the description, you must refer to this example: `imageText 2 numberSeparator 12 beforeTextDescription bla bla`. As you can see, "red" parts are customizable using the three properties described above. Furthermore, `2` is the index of the visible image, `12` the total number of images and `bla bla` the description passed thanks to the input Image object.

Attention please! There aren't spaces between values. So if you want a 'text - text' you have to specify exactly ' - ', with the two spaces
buttonsConfig
Type: ButtonsConfigoptional

Interface to configure button's visibility. All properties are optional and buttons hidden, except for the `close` button.
keyboardConfig
Type: KeyboardConfigoptional

Interface to configure keyboard's buttons. All properties are optional ad with default values. Check KeyboardConfig for more information.
enableClose Outside
Type: booleanoptional

If true, you can click on the semi-transparent background of modal images to close it. enableCloseOutside=false by default.
slideConfig
NEW
Type: SlideConfigoptional

If this object contains 'infinite: true', you won't be able to navigate before both the first and after the last images.
showDownload Button
deprecated
Type: booleanoptional

PLEASE SWITCH TO buttonsConfig. This property will be removed in version 4.0.0.

If both downloadable and showDownloadButton are true, the download button will be displayed in the modal view. Otherwise, not. downloadable=false by default
showExtUrl Button
deprecated
Type: booleanoptional

PLEASE SWITCH TO buttonsConfig. This property will be removed in version 4.0.0.

If true and the image URL is defined, the external URL button will be displayed in the modal view. Otherwise, not. downloadable=false by default
imagePointer
Type: numberoptional

Parameter useful to open the modal gallery (passing a value >=0) showing the image with index = imagePointer.

I recommend to use this input only if you want to build a custom gallery with "+n more" feature. Otherwise, you shouldn't pass this value (leave it undefined). For more information about that, I really encourage to see the official example (Array example or Observable example) to understand what I'm saying.

Also, please be careful, because I'm planning to change this feature in the next major release (still unscheduled, probably at the end of 2017).

Outputs

close

An ImageModalEvent right before that the modal view closes.

It contains an Action enum and a result (boolean or number)

  • action: Action that can be NORMAL, KEYBOARD or CLICK
  • result: true
show

An ImageModalEvent right before that the modal image will be visible.

It contains an Action enum and a result (boolean or number).

Everytime the current image changes, a new ImageModalEvent will be emitted.

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD or Action.SWIPE
  • result: number (index of an image. Not the index of the array, but incremented by 1)
firstImage

An ImageModalEvent right before that the FIRST image in modalImages is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD or Action.SWIPE
  • result: true
lastImage

An ImageModalEvent right before that the LAST image in modalImages is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD or Action.SWIPE
  • result: true
hasData

An ImageModalEvent right after the initialization of modalImages.

It contains an Action enum and a result (boolean or number)

This event is useful to know when angular-modal-gallery is ready.

  • action: Action.LOAD
  • result: true
buttonBeforeHook

A ButtonEvent right before the action related to an upper button.

It contains an Action enum, the current Image and the ButtonConfig that triggered this event

This event is a 'before hook', because you can execute your code before every action triggered by buttons. Also, since you have the full ButtonConfig in the payload, you can run different code based on button's type

  • action: Action.LOAD
  • button: ButtonConfig
  • image: Image
buttonAfterHook

A ButtonEvent right after the action related to an upper button.

It contains an Action enum, the current Image and the ButtonConfig that triggered this event

This event is an 'after hook', because you can execute your code after every action triggered by buttons. Also, since you have the full ButtonConfig in the payload, you can run different code based on button's type

  • action: Action.LOAD
  • button: ButtonConfig
  • image: Image

The payload of events related to action, emitted by this library.


action

It's the enum Action

The result/data of the emitted event. For example, it could be the index of the current image or a boolean to say if images are ready to display and so on.

DotsConfig

Interface to configure optional dots below the modal image.


visible
Type: boolean

If true, dots are visible, false otherwise. true by default.

Description

Interface to set a config image description.


strategy
Type: DescriptionStrategy

Enum to choose when description should be visible. ALWAYS_VISIBLE by default.
imageText
Type: stringoptional

It's the beginning part of the description. By default its value is the string 'Image', but you can change the language specifying your value, for instance 'Immagine' (italian), 'Bild' (german), 'Imagen' (spanish) or also an empty string ''.

Example: `imageText 2/12 - bla bla`
number Separator
Type: stringoptional

It's the text between the current image number ad the total number of images.

Example: `Image 2numberSeparator12 - bla bla`
beforeText Description
Type: stringoptional

It's the text between the total number of images and the final description text

Example: `Image 2/12beforeTextDescriptionbla bla`
customFull Description
Type: stringoptional

It's the full text description. If you set this value, all others will be ignored.

Obviously, if you want a dynamic description with the current image number and so on, you have to build it by yourself. Otherwise, you should use the other parameters of this object to customize the default description.

Size

Interface to customize images/icon sizes.


width
Type: string

It can be a valid value for the css 'width' property.

For instance, you can use either '100px', 'auto', '20%'.

Default values change depending by the related component.
height
Type: string

It can be a valid value for the css 'height' property.

For instance, you can use either '100px', 'auto', '20%'.

Default values change depending by the related component.

ButtonsStrategy

Enum to decide which buttons are visible or not. It can be one of these enum values:


  • DEFAULT: only close button
  • SIMPLE: close and download button
  • ADVANCED: close, download and exturl (to open a link in the current tab)
  • FULL: close, download, exturl (to open a link in the current tab), delete and refresh
  • CUSTOM: to add an array of ButtonConfig

ButtonType

Enum that represent the button's type. It can be one of these enum values:


  • REFRESH: refresh button
  • DELETE: delete button
  • EXTURL: extUrl button (to open a link in the current tab)
  • DOWNLOAD: download button
  • CLOSE: close button
  • CUSTOM: custom button created by you from scratch (behaviour, icons and so on)

DescriptionStrategy

Enum to decide when description should be visible or hidden. It can be one of these enum values:


  • ALWAYS_HIDDEN: description is always hidden
  • ALWAYS_VISIBLE: description is always visible
  • HIDE_IF_EMPTY: description is visible if valid

LoadingType

Enum that represents the type of the loading spinner. It can be one of these enum values:


  • STANDARD: the default spinner
  • CIRCULAR: a circular spinner
  • BARS: spinner build with some animated vertical bars
  • DOTS: spinner composed by animated dots

Action

Enum that represents the source of an action. It can be one of these enum values:


  • NORMAL: the default value
  • CLICK: mouse click action
  • KEYBOARD: keyboard action (ctrl+s, cmd+s, esc, →, ← and so on)
  • SWIPE: swipe action
  • LOAD: data/images loaded by the library

Keyboard

Enum that represents keyboard's keycodes. It can be one of these enum values:


  • ESC: 27
  • LEFT_ARROW: 37
  • RIGHT_ARROW: 39
  • UP_ARROW: 38
  • DOWN_ARROW: 40

PlainGalleryStrategy

Enum that represent the strategy used to show thumbnails in plain gallery. It can be one of these enum values:


  • ROW: to show thumbnails in a row horizontally
  • COLUMN: to show thumbnails in a row vertically
  • GRID: to show thumbnails in a grid
  • CUSTOM: to show thumbnails with a custom logic
